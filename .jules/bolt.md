## 2024-05-24 - Search Performance Optimization
**Learning:** `scoreProductMatch` recalculates regex/string normalization on every keystroke during catalog searches, causing severe main-thread blocking when combined with `.map().filter().sort().map()`. Memory mentions that search performance in `js/catalog.js` is optimized using `getNormalized(obj, key)` with a `WeakMap`, but the codebase currently lacks this helper and instead uses the expensive `normalizeSearch` repeatedly inside `scoreProductMatch`. Also, the loop allocations can be collapsed into a single pass.
**Action:** Replace `normalizeSearch` calls inside `scoreProductMatch` with a `WeakMap`-backed lazy caching mechanism (`getNormalized`). Consolidate the chained `.map().filter().sort().map()` arrays into a single imperative loop structure to avoid multiple intermediate array allocations.
