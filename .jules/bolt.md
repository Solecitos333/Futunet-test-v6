## 2024-05-24 - Lazily Caching Normalized Strings on Object Properties and Replacing Chained Array Methods
**Learning:** In a codebase heavily relying on client-side search filtering (like `js/catalog.js`), executing expensive regex replacements (`normalizeSearch`) repeatedly for every keystroke during loops is a major bottleneck. Furthermore, chained array operations (`.map().filter().sort().map()`) create unnecessary intermediate arrays, escalating memory allocation and garbage collection overhead.
**Action:** Use `Object.defineProperty` to attach non-enumerable cached versions of normalized strings directly to the target objects (`getNormalized`). This ensures string normalization is computed only once per property. Additionally, replace chained array map/filter operations with imperative `for` loops that evaluate matches, score them, and collect results in a single pass to dramatically reduce memory allocations.
