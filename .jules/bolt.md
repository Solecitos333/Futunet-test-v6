## 2026-05-10 - Lazily cache string normalization on objects
**Learning:** String normalization during search scoring (`normalizeSearch`) can be a significant bottleneck when filtering thousands of objects if properties are evaluated redundantly for every loop iteration. Furthermore, standard object assignment (`obj.cacheKey = val`) can leak into serialization or standard iteration loops.
**Action:** Use `Object.defineProperty(obj, cacheKey, { value: normalizedResult, enumerable: false })` to lazily normalize strings and cache them directly on objects without breaking application expectations. This, combined with replacing `.map().filter().sort().map()` arrays chains with imperative single-pass arrays, noticeably improves filtering execution time for large object arrays.
