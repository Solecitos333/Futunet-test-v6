## 2024-06-25 - Cache normalized strings with Object.defineProperty
**Learning:** During continuous keystroke searches on large datasets (e.g. `catalog.js` product list), standard regex replacements executed inside filtering loops can create massive CPU overhead. Caching the normalized text right on the object drastically improves performance (roughly ~4-6x improvement in mock iteration benchmarks). However, a standard assignment (`obj._norm_title = ...`) causes these new cache keys to be discovered during subsequent generic object serializations or iterations.
**Action:** Always use `Object.defineProperty(obj, cacheKey, { value: ..., enumerable: false })` when dynamically caching computed properties on data objects to guarantee the cache keys do not leak into iterators or JSON serializers, avoiding unexpected side effects while boosting performance.
