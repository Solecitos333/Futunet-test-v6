## 2024-05-18 - [Performance Optimization] Cache Normalized Strings for Search Performance
**Learning:** String normalization with regex replacements (such as stripping accents and non-alphanumeric characters) is computationally expensive, especially when recalculating it on every keystroke during an array filter loop on large datasets. Re-evaluating these properties synchronously over an array of thousands of products block the main thread and degrade the UI experience.
**Action:** Use `Object.defineProperty` with `enumerable: false` to lazily cache the output of the string normalization logic directly onto the object the first time it is accessed. This transforms expensive regex iterations into fast static lookups on subsequent keystrokes or fuzzy matches without serializing the hidden string properties.
