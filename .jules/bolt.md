## 2024-05-24 - Cached normalizeSearch calls in catalog.js
**Learning:** In a frontend catalog relying on search-as-you-type, repeated string normalizations (lowercase, replacing accents and special characters, regex replacements) during fuzzy match iterations can become a significant CPU bottleneck. Every search keystroke was previously iterating over the entire product database and re-normalizing `title`, `desc`, `category`, and `brand`.
**Action:** Implemented a lazy caching helper `getNormalized(obj, key)` which caches the normalized string as a non-enumerable property on the target object via `Object.defineProperty()`. This effectively shifts the performance characteristic from repeated O(N) regex operations per keystroke to O(1) property lookups, caching the expensive string normalization the first time it is requested.
