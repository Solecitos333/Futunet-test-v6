## 2024-05-29 - Optimize Search Normalization and Filtering
**Learning:** During text-heavy operations like search filtering on large arrays, repeatedly calling string normalization (which involves regex and string manipulation) is extremely expensive. Furthermore, chaining array methods (`.map().filter().sort().map()`) creates multiple intermediate arrays, causing unnecessary memory allocation and iteration overhead. Caching normalized strings using `Object.defineProperty` (to prevent leaking into `Object.keys` or `for...in` loops) and combining map/filter operations into a single imperative loop significantly reduces processing time (roughly 2x faster).
**Action:** When implementing filtering or scoring loops over large datasets, eagerly or lazily cache expensive transformations on the objects being iterated over using non-enumerable properties. Prefer imperative single-pass loops over multiple chained functional array methods for performance-critical paths.
