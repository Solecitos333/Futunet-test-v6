## 2025-02-12 - Imperative Loop Optimization & Property Enumeration Side-Effects
**Learning:** In hot paths like search logic where sorting and scoring occur over arrays with thousands of items, chained operations like `.map().filter().sort().map()` cause unnecessary array allocations (e.g. going from three to one intermediate allocation when optimizing). Furthermore, when dynamically caching computed string normalization using memoization straight onto product objects within iterations, you should be careful to specify `enumerable: false` using `Object.defineProperty` (e.g. `getLower(obj, key)`), so these caches don't get accidentally serialized or create unintended side-effects down the road.
**Action:** Replace multi-step array chains on large collections with single-pass imperative `for` loops. Always use `Object.defineProperty` with `enumerable: false` when dynamically adding temporary or cache values to objects that represent domain state.
