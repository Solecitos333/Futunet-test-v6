## 2024-06-19 - WeakMap Caching for Iterative Search Normalization
**Learning:** Search performance in `js/catalog.js` suffers significantly due to redundant regex mappings (`normalizeSearch`) across all products on every keystroke in filtering loops. Applying caching on object instances mutated them or broke if frozen.
**Action:** Use a `WeakMap` (`getNormalized` helper) to lazily cache and memoize normalized search strings keyed by the object references themselves. This avoids expensive operations and side-effects on the underlying data structures, cutting execution time almost in half during benchmarks. Also, replace chained array methods like `.map().filter().sort().map()` in critical search loops with an imperative single-pass `for` loop to reduce intermediate array allocations.
