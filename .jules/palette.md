## 2024-06-19 - Accessibility of dynamically generated icon-only buttons
**Learning:** Dynamically generated HTML components in JavaScript (like cart quantity controls using template literals) often lack static analysis coverage for accessibility. They can have multiple view variants (like 'compact' vs 'regular' layouts) where accessibility attributes might be applied inconsistently. In this case, `aria-label` attributes were missing on the `-` and `+` icon-only buttons in both the cart and some catalog inline controls.
**Action:** When applying UX or accessibility fixes (like adding `aria-label`s) to dynamically generated HTML components, ensure all conditional view variants are updated. Always explicitly add `aria-label` attributes to icon-only buttons during dynamic generation.
